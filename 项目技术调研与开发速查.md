# 项目技术调研 · 问答记录

---

**问：** 这是我的软件工程本科实践项目。我需要与其他四位同学协作开发。目前的难点是：我们只开发过传统的项目（vue、springboot、mysql）。对于大模型的调用以及摄像头识别这些模块，我们几乎是零经验。那么，我们第一步应该了解什么呢？

**答：** 核心就两个新领域：计算机视觉（CV）和大模型（LLM）。

CV 方面：
- 车牌识别 → YOLOv8/11（目标检测）+ PaddleOCR（文字识别），成熟开源方案
- 手势识别 → MediaPipe Hands（Google 出品），强烈推荐起步，无需训练模型，开箱即用
- 视频流处理 → OpenCV (Python) + Flask/FastAPI 做推理服务

LLM 方面：
- 调用 LLM API（DeepSeek 免费额度够用）
- Prompt Engineering（让模型输出结构化 JSON 告警信息）
- Agent 基本模式（ReAct：思考→行动→观察）

前端集成：
- WebSocket/SSE 实时推送
- Element Plus / Ant Design Vue 做监控大屏
- ECharts 告警数据可视化

关键建议：不要试图从零训练模型，预训练模型直接调用就行。

**问：** 现在重点是 cv 和 llm。我先了解一下 cv。通过你的只言片语，我大致了解到，我们的项目需要的就是目标捕捉、文字识别、手势识别，对吗？并且，这些都有现成的、开源的框架/模型对吗？

**答：** 完全正确。

| 你的说法 | CV 领域术语 | 对应模块 | 开源方案 |
|---------|------------|---------|---------|
| 目标检测 | Object Detection | 车牌定位 | YOLOv8/v11（HyperLPR3 内置） |
| 文字识别 | OCR | 车牌文字识别 | PaddleOCR / HyperLPR3 内置 |
| 手势识别 | Keypoint Detection + Gesture Classification | 车主 6 种手势 + 交警 8 种手势 | MediaPipe Hands / Pose |

关键事实：一个工具覆盖一个模块。
- 车牌识别全流程 = HyperLPR3 一家搞定，一个 `hyperlpr3.recognize(img)` 就返回车牌号 + 颜色 + 置信度。
- 车主手势 = MediaPipe Hands，21 个手部关键点，浏览器端直接跑。
- 交警手势 = MediaPipe Pose，33 个全身关键点。

三个模块，两个开源工具（HyperLPR3 + MediaPipe），全部预训练好，不需要 GPU，不需要训练数据，不需要标注。你们要做的就是"调用 API + 写规则"。

---

**问：** 请问 YOLOv8/v11、PaddleOCR、HyperLPR3 这些是"框架""技术"还是"模型（可以直接调用的）"？

**答：**

| 名称 | 本质 | 类比 |
|------|------|------|
| YOLOv8/v11 | 目标检测模型（带预训练权重） | 一个已经训练好的"眼睛"，能认出图里有什么东西在哪 |
| PaddleOCR | OCR 框架/工具包 | 一个"工具箱"，包装了多个 OCR 模型，给你统一 API |
| HyperLPR3 | 中国车牌识别专用框架 | 一个"一体机"，内置 YOLO 检测 + OCR 识别，开箱即用 |

你们只需要关心 HyperLPR3 和 MediaPipe。YOLO 和 PaddleOCR 你们根本不需要碰。

HyperLPR3 内部依赖关系：
```
hyperlpr3.recognize(img)       ← 你只需要调这一行
  ├── YOLO 做车牌检测（自动）
  └── OCR 做文字识别（自动）
  ↓
返回: {车牌号, 颜色, 置信度}
```

pip install 清单只有三个：`hyperlpr3`、`mediapipe`、`opencv-python`。

**问：** 那手势识别呢？

**答：** MediaPipe 是 Google 开源的跨平台 AI 推理框架，内置手/脸/身体等预训练模型。

| 内置模型 | 输出 | 对应需求 |
|----------|------|---------|
| Hand Landmarker | 每只手 21 个关键点的 3D 坐标 | 车主手势 |
| Pose Landmarker | 全身 33 个关键点的 3D 坐标 | 交警手势 |

调用流程：
```
MediaPipe 加载 Hand Landmarker → 喂入摄像头画面 → 返回 21 个关键点坐标
  ↓
你基于坐标写几何规则判断手势类型
  例：指尖靠近掌心 → 握拳 / 指尖远离手腕 → 手掌张开
```

关键区别：
- HyperLPR3：给你直接结果（车牌号）
- MediaPipe：只给你关键点坐标，手势分类得自己写规则（几十行 if-else）

MediaPipe 帮你干了 99% 的活（定位关键点），你只干 1%（写几何规则分类）。
