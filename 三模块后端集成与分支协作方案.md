采用 `main + develop + 三条 feature 分支`：

```text

feature/plate-backend    车牌识别
```

初始化：

```bash
git checkout main
git pull

git push -u origin develop

git checkout -b feature/gesture-backend develop
```

每天开始开发前：

```bash
git checkout develop

```

每天晚上合并：

1. 各自 push 功能分支。
2. 向 `develop` 提 PR。
3. 至少一个队友看过接口和运行结果后再合。
4. 合并后所有人重新拉取 `develop`。

Commit 规范：

```text

feat(gesture): add driver gesture command mapping
feat(alert): add websocket alert broadcast
fix(plate): handle empty recognition result
fix(gesture): reduce false trigger by duration threshold
docs(api): update backend integration contract
```

不要写 `update`、`fix bug`、`提交` 这种模糊信息。

## 6. 车牌识别接口

### 图片识别

```http
POST /api/plate/recognize-image
Content-Type: multipart/form-data
```

请求：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `file` | file | 是 | jpg/png/bmp |
| `save_record` | bool | 否 | 是否保存历史记录，默认 true |

返回 `data`：

```json
{
  "filename": "car.jpg",
  "image_size": "1920x1080",
  "inference_ms": 73.5,
  "plate_count": 1,
  "plates": [
    {
      "plate_code": "京A12345",
      "plate_color": "蓝牌",
      "confidence": 0.96,
      "bbox": [100, 220, 280, 270]
    }
  ]
}
```

### 视频识别

```http
POST /api/plate/recognize-video?interval=0.5
Content-Type: multipart/form-data
```

返回至少包含：

```json
{
  "filename": "road.mp4",
  "fps": 25.0,
  "duration_sec": 18.2,
  "processed_frames": 36,
  "unique_plates": 3,
  "plates": []
}
```

### RTSP 沙盘流识别

```http
POST /api/plate/recognize-stream
Content-Type: application/json
```

请求：

```json
{
  "rtsp_url": "rtsp://10.126.59.120:8554/live/live1",
  "duration_sec": 10,
  "sample_interval": 0.5
}
```

优先测试这些沙盘地址：

```text
rtsp://10.126.59.120:8554/live/live1   桥面
rtsp://10.126.59.120:8554/live/live2   停车场出口
rtsp://10.126.59.120:8554/live/live10  道路1
rtsp://10.126.59.120:8554/live/live11  停车场入口
```

## 7. 手势识别接口

手势模块分两类：交警手势、车主手势。若车主手势在前端用 MediaPipe JS 跑，也要把识别结果上报后端，供日志和 Agent 使用。

### 交警手势识别

```http
POST /api/gesture/traffic-police/recognize-frame
Content-Type: multipart/form-data
```

返回 `data`：

```json
{
  "gesture_type": "stop",
  "gesture_label": "停止信号",
  "confidence": 0.91,
  "keypoints": [
    {"name": "left_shoulder", "x": 0.42, "y": 0.31, "score": 0.98}
  ],
  "latency_ms": 42.3
}
```

8 种交警手势枚举：

```text
stop            停止信号
straight        直行信号
left_turn       左转弯信号
left_turn_wait  左转弯待转信号
right_turn      右转弯信号
lane_change     变道信号
slow_down       减速慢行信号
pull_over       靠边停车信号
```

### 车主手势识别

```http
POST /api/gesture/driver/recognize-frame
Content-Type: multipart/form-data
```

建议固定 6 种车主手势：

```text
open_palm      手掌张开：唤醒/主页
fist           握拳：确认/执行
circle         单指画圈：调节音量
swipe_left     左滑：上一项/上一首
swipe_right    右滑：下一项/下一首
thumb_up_down  拇指上下：接听/挂断或温度调节
```

返回 `data`：

```json
{
  "gesture_type": "swipe_right",
  "gesture_label": "右滑",
  "confidence": 0.88,
  "stable": true,
  "duration_ms": 650,
  "command": {
    "target": "media",
    "action": "next"
  },
  "hand_landmarks": []
}
```

误触发抑制规则：

1. 单帧结果不直接触发控制。
2. 同一手势连续稳定超过 `500ms` 才触发。
3. 置信度低于 `0.75` 不触发。
4. 同一指令触发后进入 `1000ms` 冷却时间。

### 前端识别结果上报

```http
POST /api/gesture/events
Content-Type: application/json
```

请求：

```json
{
  "source": "driver",
  "gesture_type": "fist",
  "gesture_label": "握拳",
  "confidence": 0.93,
  "stable": true,
  "command": {
    "target": "system",
    "action": "confirm"
  }
}
```

## 15. 最小可演示闭环

车牌 + Agent：

```text
上传道路图片/视频
  -> /api/plate/recognize-image 或 /api/plate/recognize-video
  -> 返回车牌号、颜色、置信度、bbox
  -> 后端写 plate 日志
  -> Agent 监听日志
  -> 低置信度或失败时生成告警
  -> /ws/alerts 推送到前端
  -> 前端展示识别结果和告警
```

手势 + 控车：

```text
摄像头/图片帧
  -> MediaPipe 提取关键点
  -> 分类为交警手势或车主手势
  -> 后端记录 gesture event
  -> 稳定车主手势映射为车辆控制指令
  -> 异常跳变或低置信度触发 Agent 告警
```

## 16. 风险备用方案

| 风险 | 处理方式 |
| --- | --- |
| 车牌视频流太慢 | 先抽帧识别，每 0.5-1 秒识别一帧；演示时用短视频 |
| RTSP 沙盘连不上 | 使用本地视频替代，同时保留 RTSP 接口和错误告警演示 |
| 手势分类不稳定 | 先保证车主 6 种手势和 3-4 类交警手势可演示，再补齐 8 类 |
| LLM API 不稳定 | Agent 增加规则降级摘要，不依赖 LLM 也能生成告警 |
| 合并冲突 | 公共文件集中由一人改，三个人不要同时改 `main.py` 和 `requirements.txt` |
| 时间不够 | 优先保证后端 API、日志、Swagger、最小闭环，前端美化后置 |
