## 2026-07-10 追加记录：左右滑与下滑误触优化

本次在“上下滑动控制音量”的基础上，继续优化车主端左右滑识别稳定性。

### 修改内容

- 文件：`vendor/web_gesture_backend/dgcore/utils/action_controller.py`
- 给左右滑增加了基于手部中心点水平位移的兜底识别。
- 修正 `hand_right / hand_left` 在二号、三号起点时落到错误终点枚举的问题。
- 根据日志中右滑过程更容易被识别成其他手势的问题，将 `call(20)`、`middle_finger(23)`、`like(27)` 加入右滑候选判别；这些候选需要配合中心点确实向右移动，才会作为右滑链条的一部分。
- 进一步按整份 `gesture_static_trace.log` 统计右移窗口：右滑起点高频为 `unknown / three_gun / like / call / two_left`，右滑终点高频为 `unknown / hand_right / like / peace_inverted / two_left / grabbing / half_right / peace`。已将右滑候选拆成起点候选与终点候选，终点必须满足实际向右位移和最短帧数后才触发。
- 左右滑的中心点水平位移判定保留为兜底，不作为优先主判据；优先通过状态机补全右滑过程中常见的误识别标签。
- 已撤回对 `FAST_SWIPE_UP / FAST_SWIPE_DOWN` 的额外纵向位移约束，恢复上下滑音量调节的原有识别宽松度。
- 左右滑识别层不加冷却，保持和上下滑一样的原始动作上报逻辑；冷却只应放在具体业务动作层，例如音量调节或后续翻页执行。
- 增加 `three_gun(41)` 与 `unknown(-1/None)` 之间跳跃的窄范围平滑：同一条手部轨迹最近 6 帧内只能出现 `three_gun/unknown`，且 `three_gun` 超过一半时，才把当前 `unknown` 平滑为 `three_gun`。

### 日志观察

查看 `logs/gesture_static_trace.log` 后发现：

```text
SWIPE_LEFT: 51
SWIPE_RIGHT: 15
FAST_SWIPE_DOWN: 28
FAST_SWIPE_UP: 21
```

左滑识别率明显高于右滑，主要原因是模型在测试中更常输出 `hand_right`，且右滑过程中更容易出现 `unknown / three_gun / like / call / middle_finger` 等中间标签，导致轨迹不如左滑连续。

左右滑测试时偶尔触发下滑的原因是 `FAST_SWIPE_DOWN` 原规则偏宽松，手势标签抖动叠加检测框纵向抖动时会误判为快速下滑。

后续原则：不在识别层牺牲上下滑识别率来压误触；识别层保持动作输出，具体业务层根据音量调节、翻页等动作分别做冷却/防抖。

保持某个手势时出现 `three_gun / unknown` 来回跳，主要是静态分类器对该手型边界不稳定。已采用单独的原始标签历史窗口做平滑，避免动作状态机 `clear()` 清空历史后导致防抖失效。

no_smooth_half= [41, -1, 41, -1]
