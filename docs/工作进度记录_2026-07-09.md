### 2. 后端接口

当前主要接口包括：

- `GET /api/health`：后端健康检查。
- `GET /api/cameras`：获取沙盘摄像头列表。
- `GET /api/cameras/{id}/stream`：将 RTSP 转成网页可访问的 MJPEG 流。
- `GET /api/cameras/{id}/snapshot.jpg`：获取摄像头快照。
- `POST /api/plate/recognize-image`：图片车牌识别。
- `POST /api/plate/recognize-video`：视频车牌识别。
- `POST /api/plate/recognize-stream`：RTSP 视频流车牌识别。
- `POST /api/gesture/start`：启动手势识别。
- `POST /api/gesture/stop`：停止手势识别。
- `GET /api/gesture/video-feed`：手势识别画面流。
- `POST /api/gesture/events`：模拟/上报手势事件。
- `GET /api/alerts`：获取 Agent 告警。
- `POST /api/logs/simulate`：模拟异常日志并触发 Agent 告警。
- `WS /ws/alerts`：告警 WebSocket。

## 当前已知问题

### 1. 沙盘 RTSP 流当前不可用

已批量测试 12 路 RTSP：

```text
live1-live12 全部 opened=False
FFmpeg 日志：method DESCRIBE failed: 404 Not Found
```

结论：

- 后端服务没有崩。
- 前端也没有坏。
- 当前问题是 `10.126.59.120:8554` 上没有发布 `/live/live1` 到 `/live/live12` 这些 RTSP 流，或者推流端已经换了 IP/路径。

页面中出现：

```text
Camera stream unavailable
reconnecting
```

这是后端生成的兜底画面，表示它正在持续重连。

需要后续确认：

- 沙盘推流端 MediaMTX 是否启动。
- 12 路视频是否正在推流。
- 当前电脑是否在能访问 `10.126.59.120` 的同一网络。
- 老师/同学是否更换了推流 IP、端口或路径。

## 设计方向

当前产品方向已经确定为“双端系统”。

### 管理者端

应用场景：

- 停车场管理
- 交通管理
- 违章线索记录

课程条件限制下，管理者端优先只做：

- 多路视频流切换
- 自动车牌识别
- 识别结果记录
- Agent 告警提示

### 车主端

应用场景：

- 驾驶辅助
- 违章举报辅助
- 交警手势理解
- 车主手势控车
- Agent 安全预警

当前硬件限制下：

- 外置车外摄像头暂时用网站里的沙盘视频流代替。
- 内置车内摄像头用本机前置摄像头代替。
